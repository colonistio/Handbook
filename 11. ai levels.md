# AI Proficiency Levels

## About

This document defines five levels of AI proficiency.

Every team member should operate at **Level 4 (Power User)** or above.

**References:**
+ [larridin.com/blog/what-is-ai-proficiency](http://larridin.com/blog/what-is-ai-proficiency)
+ [co-r-e.com/method/ai-skill-levels](http://co-r-e.com/method/ai-skill-levels)

Read to better understand each level and learn how to improve. This doc is only the start point.

## Levels Overview

| Level | Name | Description |
| :---- | :---- | :---- |
| **1** | **Search Replacer** | Uses AI as a better Google. Single-shot queries. |
| **2** | **Task Automator** | AI for specific repetitive tasks. Better context but transactional. |
| **3** | **Augmented Worker** | AI is a thinking partner. Multi-turn, multi-model, diverse use cases. |
| **4** | **Power User** | Custom instructions, APIs, reusable workflows embedded in daily work. |
| **5** | **AI-Native Orchestrator** | Delegates complex multi-step work to autonomous AI systems. |

## Level Definitions

### Level 1: Search Replacer

*Uses AI as a better Google. Single-shot questions, one tool, gives up when the first output isn't right.*

**Examples:**
+ Asks "Write a PRD for a new feature" with no context about the product, users, or constraints — then uses the generic output or decides AI is useless.
+ Pastes a bug report into ChatGPT with just "what's wrong?", reads the first suggestion, doesn't follow up or refine.

### Level 2: Task Automator

*Uses AI for specific repetitive tasks. Better prompts, some refinement, but still transactional and limited to a few use cases.*

**Examples:**
+ Uses AI to draft release notes or rewrite user-facing copy with decent context (audience, tone, length) — but only ever for this one task.
+ Asks AI to summarize a user research transcript but doesn't then use it to extract patterns, generate insights, or draft recommendations.

### Level 3: Augmented Worker

*AI is a genuine thinking partner. Multi-turn conversations, multiple tools, used across research, writing, analysis, and planning.*

**Examples:**
+ Before writing a feature spec, have a 10-turn conversation with AI: shares product context, discusses user needs, iterates on scope and edge cases, stress-tests trade-offs, then writes the spec with that thinking as the foundation.
+ After a sprint, feed ticket data and retro notes into AI to surface patterns: "What types of work keep slipping? Where are we underestimating?" then brings insight to the team.

### Level 4: Power User

*Fluent across models and tools. Custom instructions, reusable workflows, and AI integrations embedded in how they work — not just what they do.*

**Examples:**
+ Has a pre-loaded AI project with product context, design principles, and past decisions — so every conversation starts with rich context instead of from scratch.
+ Builds a reusable prompt template for competitor analysis or feature evaluation that any team member can run and get structured, consistent output.

### Level 5: AI-Native Orchestrator

*Delegates complex, multi-step work to AI systems that execute autonomously. Reimagines what one person can accomplish.*

**Examples:**
+ Designs an end-to-end workflow: user feedback comes in → AI categorizes and clusters it → AI maps clusters to product areas → AI drafts a prioritized insights summary → human reviews and acts. The chain runs from a single trigger.
+ Instead of doing 20 tasks manually, designs 5 AI systems that handle the execution layer, freeing their time for judgment, strategy, and stakeholder work.

## How to Level Up

### Level 1 → 2: Stop accepting the first answer
+ Never end an AI conversation after one exchange. Always do at least 3 rounds of refinement.
+ Before prompting, add context: who is this for, what's the goal, what format, what tone.

### Level 2 → 3: Make AI your thinking partner
+ Use AI for tasks you haven't tried yet: brainstorming, analysis, stress-testing your own ideas.
+ Try a second AI tool. Compare outputs. Share documents and prior work for richer context.

### Level 3 → 4: Build systems, not one-offs
+ Identify your 3 most repeated tasks. Write a reusable prompt template or custom instruction set for each.
+ When you find a great workflow, document it and share it with the team.

### Level 4 → 5: Automate the execution layer
+ What multi-step process could you design once and let AI run repeatedly?
+ Redesign your role around what AI can handle. Focus human time on judgment and strategy.
